# Persuasion Principles for Discipline Skills

LLMs respond to the same persuasion principles as humans. Understanding this helps design more effective discipline-enforcing skills.

**Research:** Meincke et al. (2025) tested 7 principles with N=28,000 AI conversations. Persuasion techniques more than doubled compliance (33% → 72%, p < .001).

## Principles by Use Case

| Skill Type | Use | Avoid |
|------------|-----|-------|
| Discipline-enforcing | Authority + Commitment + Social Proof | Liking, Reciprocity |
| Technique / guidance | Moderate Authority + Unity | Heavy authority |
| Reference | Clarity only | All persuasion |

## Authority

Deference to expertise, credentials, official sources.

- Imperative language: "YOU MUST", "Never", "Always"
- Non-negotiable framing: "No exceptions"
- Eliminates decision fatigue and rationalization

```markdown
✅ Write code before test? Delete it. Start over. No exceptions.
❌ Consider writing tests first when feasible.
```

## Commitment

Consistency with prior actions or declarations.

- Require announcements: "Announce skill usage"
- Force explicit choices: "Choose A, B, or C"
- Use tracking: TodoWrite for checklists

## Social Proof

Conformity to what others do.

- Universal patterns: "Every time", "Always"
- Failure modes: "X without Y = failure"

## Scarcity

Urgency from time limits.

- Time-bound: "Before proceeding"
- Sequential: "Immediately after X"

## Unity

Shared identity, "we-ness".

- Collaborative: "our codebase", "we're colleagues"
- Shared goals: "we both want quality"

## What NOT to Use

- **Liking** — creates sycophancy, conflicts with honest feedback
- **Reciprocity** — rarely needed, can feel manipulative

## How to Apply

1. **What type is the skill?** (Discipline vs Technique vs Reference)
2. **What behavior am I changing?**
3. **Which principle(s) apply?** (Usually authority + commitment for discipline)
4. **Am I combining too many?** (Don't use all seven)
5. **Is this ethical?** (Serves user's genuine interests?)

## Ethical Test

Would this technique serve the user's genuine interests if they fully understood it?

- **Legitimate:** Ensuring critical practices are followed, preventing predictable failures
- **Illegitimate:** Manipulating for personal gain, creating false urgency, guilt-based compliance
